# データフロー設計

## 日次更新フロー

1. Vercel Cron 起動
2. 対象市場 / 対象 symbol を取得
3. Stooq API key を使って外部データ取得
4. データ整形
5. 市場 / 通貨 / sourceSymbol を付与
6. 最新データ保存
7. 取り込み失敗銘柄の状態保存
8. 候補計算
9. 通知判定
10. Push 通知送信

## クライアント起動時フロー

1. PWA 起動
2. IndexedDB から前回データ読込
3. 画面表示
4. 最新データ確認
5. 差分更新
6. 再計算
7. 画面反映

## 検索フロー

1. 市場を選択
2. 銘柄コードまたは銘柄名入力
3. ローカルマスタ検索
4. 結果一覧表示
5. 詳細遷移

## スクリーニングフロー

1. 条件指定
2. 条件適用
3. 指標計算
4. ランキング計算
5. 候補一覧表示

## シミュレーションフロー

1. 候補銘柄選択
2. 購入価格入力
3. 株数入力
4. 購入金額算出
5. 構成比再計算
6. 円グラフ更新
7. 改善コメント生成

## 保有登録フロー

1. 銘柄詳細またはポートフォリオから保有登録へ遷移
2. 対象銘柄と市場を確認
3. 保有株数を入力
4. 取得単価を入力
5. 通貨と現金メモを確認
6. IndexedDB に保有情報を保存
7. ポートフォリオへ戻る
8. 構成比と含み損益を再計算
