# データフロー設計

## 日次更新フロー

1. Vercel Cron 起動
2. 日本 / 米国 / 英国 / 香港 / 為替の対象単位を判定
3. Stooq API key を使って日足を取得
4. bulk 取得が使える市場では daily ASCII bulk を取得
5. bulk 取得が使えない市場では対象銘柄マスタに対して個別 CSV を取得
6. 株式 / ETF / REIT のみを保存対象にする
7. 先物 / オプション / 債券 / 指数 / 暗号資産 / 派生的な商品カテゴリを除外する
8. 為替は `USDJPY` / `GBPJPY` / `HKDJPY` を取得する
9. データ整形
10. 市場 / 商品種別 / 通貨 / sourceSymbol を付与
11. 最新データ保存
12. 空の価格ファイルは価格データなし状態として保存
13. 取り込み失敗銘柄の状態保存
14. 為替取得失敗ペアの状態保存
15. 候補計算
16. 通知判定
17. Push 通知送信

## 手動更新フロー

1. ホームまたは銘柄詳細から手動更新を実行
2. 対象銘柄または失敗マーク付き銘柄を判定
3. Stooq 個別 CSV endpoint で日足を取得
4. 取得成功時は日次価格を保存し、失敗マークを解除
5. 取得失敗時は失敗マークと理由を更新
6. 価格データが空の場合はエラーではなく価格データなし状態として表示

## クライアント起動時フロー

1. PWA 起動
2. IndexedDB から前回データ読込
3. 画面表示
4. 最新データ配信 API に更新有無を問い合わせ
5. サーバー側に新しい市場データがあれば取得
6. 取得した市場データを、その端末の IndexedDB に保存
7. 差分更新
8. 再計算
9. 画面反映

## 保存先の考え方

- Vercel Cron はサーバー側で実行され、ユーザー端末の IndexedDB には直接書き込まない
- PWA / Web は、開かれた実行環境ごとの IndexedDB に保存する
- Mac のブラウザ、スマホのブラウザ、ホーム画面に追加した PWA は、それぞれ別の IndexedDB を持つ前提で設計する
- 市場データはサーバー側の保存先から配信し、各端末の IndexedDB はローカルキャッシュとして使う
- 保有情報は MVP では IndexedDB に保存し、複数端末同期は後続のサーバー保存で扱う

## 検索フロー

1. 市場を選択
2. 銘柄コードまたは銘柄名入力
3. ローカルマスタ検索
4. 結果一覧表示
5. 詳細遷移

## スクリーニングフロー

1. 条件指定
2. 条件適用
3. 指標計算
4. ランキング計算
5. 候補一覧表示

## シミュレーションフロー

1. 候補銘柄選択
2. 購入価格入力
3. 株数入力
4. 購入金額算出
5. 構成比再計算
6. 円グラフ更新
7. 改善コメント生成

## 保有登録フロー

1. 銘柄詳細またはポートフォリオから保有登録へ遷移
2. 対象銘柄と市場を確認
3. 保有株数を入力
4. 取得単価を入力
5. 通貨と現金メモを確認
6. IndexedDB に保有情報を保存
7. ポートフォリオへ戻る
8. 構成比と含み損益を再計算

## ポートフォリオ評価フロー

1. 保有銘柄を読み込む
2. 各銘柄の最新日足を取得する
3. 外貨建て保有の通貨を判定する
4. 必要な為替ペアの日足を取得する
5. JPY 換算評価額を算出する
6. 現金込み構成比を算出する
7. 為替レート不足があれば評価不能状態を表示する
