# データフロー設計

## 日次更新フロー

1. Vercel Cron 起動
2. 日本 / 米国 / 英国 / 香港 / 為替の対象単位を判定
3. Stooq API key を使って日足を取得
4. bulk 取得が使える市場では daily ASCII bulk を取得
5. bulk 取得が使えない市場では対象銘柄マスタに対して個別 CSV を取得
6. 株式 / ETF / REIT のみを保存対象にする
7. 先物 / オプション / 債券 / 指数 / 暗号資産 / 派生的な商品カテゴリを除外する
8. 為替は `USDJPY` / `GBPJPY` / `HKDJPY` を取得する
9. データ整形
10. 市場 / 商品種別 / 通貨 / sourceSymbol を付与
11. 最新データ保存
12. 空の価格ファイルは価格データなし状態として保存
13. 取り込み失敗銘柄の状態保存
14. 為替取得失敗ペアの状態保存
15. 候補計算
16. 通知判定
17. Push 通知送信

## クライアント起動時フロー

1. PWA 起動
2. IndexedDB から前回データ読込
3. 画面表示
4. 最新データ確認
5. 差分更新
6. 再計算
7. 画面反映

## 検索フロー

1. 市場を選択
2. 銘柄コードまたは銘柄名入力
3. ローカルマスタ検索
4. 結果一覧表示
5. 詳細遷移

## スクリーニングフロー

1. 条件指定
2. 条件適用
3. 指標計算
4. ランキング計算
5. 候補一覧表示

## シミュレーションフロー

1. 候補銘柄選択
2. 購入価格入力
3. 株数入力
4. 購入金額算出
5. 構成比再計算
6. 円グラフ更新
7. 改善コメント生成

## 保有登録フロー

1. 銘柄詳細またはポートフォリオから保有登録へ遷移
2. 対象銘柄と市場を確認
3. 保有株数を入力
4. 取得単価を入力
5. 通貨と現金メモを確認
6. IndexedDB に保有情報を保存
7. ポートフォリオへ戻る
8. 構成比と含み損益を再計算

## ポートフォリオ評価フロー

1. 保有銘柄を読み込む
2. 各銘柄の最新日足を取得する
3. 外貨建て保有の通貨を判定する
4. 必要な為替ペアの日足を取得する
5. JPY 換算評価額を算出する
6. 現金込み構成比を算出する
7. 為替レート不足があれば評価不能状態を表示する
