# 通知設計

## 通知種別

### 1. 日次更新通知
- データ更新完了
- 今日の候補 TOP3

### 2. ウォッチ通知
- 条件一致
- 高値接近
- ランキング上昇

## 通知設定

- 通知 ON / OFF
- 種別ごとの ON / OFF
- 通知時刻
- 候補通知件数

## Push 購読フロー

1. 通知許可要求
2. Push Subscription 取得
3. サーバーへ保存
4. 通知種別設定保存

## 送信フロー

1. cron 実行
2. 候補 / 条件判定
3. 送信対象抽出
4. Push 送信

## MVP 方針

- 通知は個人利用を前提とする
- 複雑なセグメント配信はしない
- 日次通知とウォッチ通知を優先する
