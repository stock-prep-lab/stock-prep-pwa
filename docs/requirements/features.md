# 機能要件

## 1. ホーム

### 概要
アプリを開いた時点で、今日見るべき候補とポートフォリオの状態を把握できる。

### 表示内容
- 最終更新日時
- 更新状態
- 今日の候補 TOP
- ポートフォリオ概要
- リバランス注意
- ウォッチ銘柄変化

## 2. 銘柄検索

### 概要
銘柄コードまたは銘柄名で検索できる。

### 入力
- 銘柄コード
- 銘柄名
- 市場

### 出力
- 銘柄一覧
- 市場
- 通貨
- 終値
- 前日比
- 簡易トレンド状態

## 3. 銘柄詳細

### 概要
個別銘柄の価格情報、チャート、トレンド分析を表示する。

### 表示内容
- 基本情報
- 市場
- 通貨
- 終値
- 前日比
- 高値 / 安値
- 出来高
- 52週高値 / 安値
- ローソク足チャート
- 25MA / 75MA
- トレンド分析
- コメント

### 操作
- ウォッチ追加
- 保有追加 / 編集
- 購入シミュレーション遷移

## 4. スクリーニング

### 概要
条件を指定して候補銘柄を抽出する。

### 条件例
- 25MA より上
- 75MA より上
- 25MA 上向き
- 高値接近率
- 出来高倍率
- 流動性条件

### 出力
- 候補一覧
- 候補理由
- 並び替え結果

## 5. 候補ランキング

### 概要
候補銘柄をモメンタムベースで順位付けする。

### MVP 方針
- 12-1 モメンタム
- 6-1 モメンタム
- 高値接近率
- 流動性補助

## 6. ポートフォリオ管理

### 概要
保有登録画面から、保有銘柄、取得単価、株数、現金を管理する。

### 入力
- 銘柄コード
- 市場
- 通貨
- 保有株数
- 取得単価
- 現金

### 操作
- 保有追加
- 保有編集
- 保有一覧から銘柄詳細へ遷移

### 出力
- 総評価額
- 含み損益
- 構成比
- 現金比率
- 上位集中度
- 外貨建て保有の JPY 換算評価額

## 7. リバランス提案

### 概要
現在の保有に対して改善余地のある候補銘柄を提案する。

### MVP 方針
- 最小分散寄りの改善度
- 集中度緩和
- 現金活用
- 候補追加前後の差分提示

## 8. 購入シミュレーション

### 概要
購入価格と株数を入力し、購入後の構成変化を表示する。

### 入力
- 対象銘柄
- 購入価格
- 株数

### 出力
- 購入金額
- 現金残高変化
- 構成比変化
- 改善コメント
- 円グラフ Before / After

## 9. ウォッチリスト

### 概要
気になる銘柄を登録し、継続監視できる。

### 表示内容
- 銘柄名
- 終値
- 前日比
- 簡易トレンド状態
- 通知状態

## 10. Push 通知

### 概要
日次更新、候補通知、ウォッチ通知を行う。

### MVP 通知
- データ更新完了
- 今日の候補 TOP3
- ウォッチ銘柄の条件一致

## 11. データ同期

### 概要
日次で最新データを取り込み、ローカルデータを更新する。

### 方針
- 外部データソースは Stooq CSV を第一候補にする
- Stooq CSV 取得は API key が必要になる前提で扱う
- MVP の主対象は日本株とし、米国株、英国株、香港株の日足も同じ取り込み設計に含める
- 米国株、英国株、香港株のポートフォリオ評価用に為替日次も取り込む
- 市場ごとに Stooq 用 symbol suffix を保持する
  - 例: 米国株 `.US`
  - 例: 英国株 `.UK`
  - 例: 香港株 `.HK`
- 為替は Stooq の通貨ペア symbol を保持する
  - 例: `USDJPY`
  - 例: `GBPJPY`
  - 例: `HKDJPY`
- Stooq で取得できない銘柄や市場は、未対応として画面に明示できるようにする
- Vercel Cron による定期実行
- 起動時に最新データ確認
- ローカル保存更新

## 12. レスポンシブ UI

### 概要
主要画面がモバイル幅とデスクトップ幅の両方で成立すること。

### 対象
- ホーム
- 検索
- スクリーニング
- ポートフォリオ
- 銘柄詳細
- リバランス提案
- 購入シミュレーション

### 完了条件
- 主要情報が欠けない
- 主要ボタンが隠れない
- 不要な横スクロールが発生しない
- チャートや円グラフが視認可能な大きさを保つ
